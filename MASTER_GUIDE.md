### If Demo Fails:
1. **Stay calm** - Don't panic!
2. **Use screenshots** - Have them ready on desktop
3. **Explain what would happen** - Walk through the process
4. **Show code instead** - "Let me show you how it works..."
5. **Recovery is impressive** - Professors value problem-solving!

### If You Blank:
1. **Look at Quick Reference** - That's why you printed it!
2. **Go to anchor slide** - Model Comparison table
3. **Use time-buy phrases** - "Let me rephrase that..."
4. **Take a breath** - 2-second pause is fine
5. **Keep going** - Don't apologize, just continue

### If Question Stumps You:
1. **Don't make up answers** - Professors can tell!
2. **Use honest phrases:**
   - "That's an excellent area for future research"
   - "My current understanding is... but I'd need to verify"
   - "Great question - I'd love to explore that more after class"
3. **Redirect if possible** - "What I can tell you is..."
4. **Stay confident** - Not knowing ONE thing is fine!
